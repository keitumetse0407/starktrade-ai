# StarkTrade Platform Audit

Generated by parallel agents pattern.

## Service Boundaries + Ports
- Frontend: 3000 (Next.js 16.2.2)
- Backend: 8000 (FastAPI)
- Trading Engine: 8081 (Go)
- PostgreSQL: 5432
- Redis: 6379
- Qdrant: 6333

## Data Flow
1. Frontend → Backend API (port 8000) → Database/Redis
2. Trading Engine (8081) ↔ Alpaca API ↔ Market
3. Celery workers → Background tasks → Redis queue

## HIGH-RISK AREAS (Top 10)
1. Backend auth endpoints — JWT handling
2. Trade execution endpoints — Alpaca integration
3. Payment endpoints — Paystack integration
4. Environment variables — API keys exposure
5. Order validation — client-side vs server-side
6. WebSocket live data — real-time price feed
7. Database migrations — schema changes
8. Celery tasks — async trade execution
9. Docker secrets — production keys
10. Rate limiting — external APIs
